# SaaS后台管理系统 - 架构说明

## 文件结构

```
ar-upi-site/
├── main.html                      # 主框架页面（入口文件）
├── common.css                     # 共享CSS样式文件
├── bank_card_management.html      # 银行卡管理页面
├── fund_change.html               # 资金账变页面
├── member_list.html               # 会员列表页面
├── member_level.html              # 会员等级管理页面
├── member_tags.html               # 会员标签管理页面
├── blacklist.html                 # 黑名单管理页面
├── deposit_records.html           # 充值记录页面
├── withdraw_records.html          # 提现记录页面
└── saas_complete_member_system.html # 旧版单页面系统（已废弃）
```

## 架构特点

### 1. 模块化设计
- 每个功能页面独立为一个HTML文件
- 便于维护和开发
- 多人协作时不会产生冲突

### 2. 主框架 (main.html)
- **功能**：
  - 顶部导航栏
  - 页签管理系统
  - iframe内容加载
  - 默认打开所有页签
  
- **特性**：
  - 支持多页签切换
  - 支持页签右键菜单（关闭、刷新等）
  - 自动记忆已打开页签
  - 下拉菜单延迟1秒隐藏

### 3. 共享样式 (common.css)
- 统一的UI风格
- 所有子页面引用同一个CSS文件
- 修改样式时只需修改一个文件
- 包含以下组件样式：
  - 表格样式（包括固定列）
  - 按钮样式
  - 表单样式
  - 标签样式
  - 开关按钮
  - 分页组件
  - 统计卡片

### 4. 子页面
每个子页面都是独立的HTML文件，包含：
- 页面标题
- 筛选条件
- 数据表格
- 操作按钮
- JavaScript交互逻辑

## 使用方法

### 启动系统
1. 直接打开 `main.html` 文件
2. 系统会自动加载所有子页面到页签栏
3. 点击页签切换不同功能页面

### 页签操作
- **左键点击**：切换到对应页面
- **右键点击**：显示上下文菜单
  - 刷新当前页面
  - 关闭当前页签
  - 关闭其他页签
  - 关闭所有页签
  - 关闭左侧/右侧页签

### 顶部菜单
- **会员管理**：会员列表、等级、标签、银行卡、黑名单
- **代理管理**：（待开发）
- **活动管理**：（待开发）
- **报表**：充值记录、提现记录、资金账变
- **设置**：（待开发）

## 开发建议

### 添加新页面
1. 创建新的HTML文件
2. 引用 `<link rel="stylesheet" href="common.css">`
3. 按照现有页面结构编写代码
4. 在 `main.html` 中添加导航菜单项和默认页签配置

### 修改样式
- 全局样式修改 `common.css`
- 页面特有样式在页面内添加 `<style>` 标签

### 调试
- 使用浏览器开发者工具
- 每个页面都可以单独打开调试
- 主框架中的页面在iframe中运行

## 与旧版对比

| 特性 | 旧版 (saas_complete_member_system.html) | 新版 (模块化架构) |
|------|----------------------------------------|------------------|
| 文件数量 | 1个大文件 (3000+行) | 多个小文件 (每个200-400行) |
| 维护难度 | 困难 | 简单 |
| 团队协作 | 容易冲突 | 互不影响 |
| 加载速度 | 慢 | 快（按需加载） |
| 代码复用 | 低 | 高（共享CSS） |
| 可扩展性 | 差 | 好 |

## 后续优化建议

1. **添加路由管理**：支持URL参数，刷新页面时保持当前页签
2. **状态管理**：使用LocalStorage保存用户偏好设置
3. **权限控制**：根据用户角色显示不同菜单
4. **懒加载**：只加载当前激活的页签内容
5. **API集成**：连接后端接口，实现真实数据交互
6. **响应式优化**：适配移动端显示
7. **主题切换**：支持深色模式
8. **国际化**：支持多语言切换

- **iframe**：页面隔离和加载
